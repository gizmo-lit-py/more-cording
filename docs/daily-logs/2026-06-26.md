# 今日やったこと：PHPとMySQLをPDOで接続する準備

今日は、PHPからMySQLを使うための基本的な流れを学習した。

まず、`php-mysql-memo` という練習用フォルダを作り、`public`、`src`、`sql` に分けて構成した。

* `public`：ブラウザからアクセスするPHPファイルを置く場所
* `src`：DB接続など、裏側のPHP処理を書く場所
* `sql`：DBやテーブルを作るためのSQLファイルを置く場所

`sql/schema.sql` には、`php_mysql_memo` というデータベースと、`memos` テーブルを作るSQLを書いた。

その後、HomebrewでMySQLをインストールし、`brew services start mysql` でMySQLサーバーを起動した。
`mysql -u root < sql/schema.sql` を実行して、SQLファイルの内容をMySQLに反映した。

`SHOW TABLES` や `DESCRIBE memos` で確認したところ、`memos` テーブルが作成されていて、`id`、`content`、`created_at` のカラムも確認できた。

次に、`src/db.php` にPDO接続用の関数 `getPdo()` を作成した。

PDOは、PHPからMySQLに接続してSQLを送るための接続オブジェクトだと理解した。
`$dsn` には、MySQLの接続先として `host=localhost`、DB名として `php_mysql_memo`、文字コードとして `utf8mb4` を指定した。

また、MySQLの `root` ユーザーは管理者ユーザーで、今回はローカル学習環境なのでパスワードなしの空文字で接続した。

`public/db_test.php` では、`require_once` で `src/db.php` を読み込み、`getPdo()` を呼び出してMySQLに接続した。
ブラウザで `db_test.php` を開いたところ、「DB接続成功」と表示され、PHPからPDO経由でMySQLに接続できることを確認できた。

今日理解したことは、`schema.sql` はデータベース本体ではなく、DBやテーブルを作るための設計図であり、実際のデータはMySQLサーバーの中に保存されるということ。
また、`db.php` はそのMySQLにPHPから接続するためのファイルで、PDOオブジェクトを作って返す役割を持っている。

次回は、PDOを使って `memos` テーブルからデータを取得し、ブラウザに表示するところを進める。
